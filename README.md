# 📝 Interactive To-Do List

## 📌 About the Project
This project is an interactive web application for managing daily tasks (To-Do List). It was designed and developed as a practical, intensive training on the fundamentals and core concepts of **JavaScript**, specifically focusing on DOM Manipulation and State Management.

📅 **Created:** June 2026  

---

## 🚀 Features
- **Create Tasks:** Add new tasks with a title, description, and priority level (Urgent, High, Medium, Low).
- **Update Tasks:** Retrieve any task's data back into the form to edit it seamlessly without losing its order.
- **Delete Tasks:** Remove tasks from the list with an instant UI update.
- **Mark as Completed:** Toggle the task status and dynamically change its visual styling to distinguish it from pending tasks.
- **Local Storage:** All tasks are automatically saved in the user's browser, ensuring data is never lost upon refreshing or closing the page.
- **Interactive UI:** - Floating labels for input fields that adapt when text is entered.
  - Dynamic hiding and showing of forms for a smooth User Experience (UX).
- **Roadmap:** Future plans include adding a Calendar feature to schedule and track tasks chronologically ⏳.

---

## 🛠️ Technologies Built With
- **HTML5:** For structuring the core layout and forms.
- **CSS3:** For styling the interface and adding visual effects (including FontAwesome for icons).
- **Vanilla JavaScript (ES6+):** The core engine of the project, built entirely without frameworks to solidify the following concepts:
  - Creating DOM elements programmatically (`createElement`, `appendChild`).
  - Event management (`EventListeners`, `Event Object`).
  - Local Storage & data formatting (`JSON.stringify`, `JSON.parse`).
  - Advanced array methods (`filter`, `splice`).

---

## 📂 Folder Structure
- `index.html`: The main page containing the task entry form and the task list table.
- `Completed.html` / `To-Do.html`: Pages dedicated to task filtering (currently under development).
- `Calendar.html`: The future calendar page.
- `CSS/`: Contains the project's stylesheets.
- `Js/script.js`: The heart of the project, containing cleanly separated JS logic (Clean Code) dividing data state management and DOM manipulation.

---

## 💡 Learning Outcomes
Throughout the development of this project, I actively practiced and mastered:
1. The clear separation of concerns between "Application Data" (Array of Objects) and "Visual Presentation" (HTML Table).
2. Preventing default form submission behaviors using `e.preventDefault()`.
3. Dynamically binding events and functions inside loops using the task's `Index`.
4. Solving data synchronization issues and avoiding programmatic class conflicts.

# 📝 تطبيق قائمة المهام التفاعلي (Interactive To-Do List)

## 📌 نبذة عن المشروع
هذا المشروع عبارة عن تطبيق ويب تفاعلي لإدارة المهام اليومية، تم تصميمه وتطويره كتدريب عملي ومكثف على أساسيات ومفاهيم لغة **JavaScript**، وتحديداً التلاعب بعناصر الصفحة (DOM Manipulation) وإدارة حالة البيانات (State Management).

📅 **تاريخ الإنشاء:** يونيو 2026  

---

## 🚀 المميزات
- **إضافة المهام (Create):** إضافة مهمة جديدة تتضمن عنوان، وصف، ومستوى الأهمية (عاجل، عالي، متوسط، منخفض).
- **تعديل المهام (Update):** استرجاع بيانات أي مهمة إلى النموذج (Form) وتعديلها بسهولة دون فقدان الترتيب.
- **حذف المهام (Delete):** إزالة المهام من القائمة مع تحديث فوري للواجهة.
- **تحديد المهام المنجزة (Mark as Completed):** تغيير حالة المهمة وشكلها بصرياً لتمييزها عن المهام غير المنجزة.
- **التخزين المحلي (Local Storage):** جميع المهام تُحفظ تلقائياً في متصفح المستخدم، مما يضمن عدم ضياع البيانات عند تحديث أو إغلاق الصفحة.
- **واجهة مستخدم تفاعلية (Interactive UI):** - تأثيرات حركية للحقول (Floating Labels) عند إدخال النصوص.
  - إخفاء وإظهار النماذج ديناميكياً لتجربة مستخدم سلسة (UX).
- **خطط مستقبلية (Roadmap):** إضافة ميزة التقويم (Calendar) لجدولة المهام ومتابعتها زمنياً ⏳.

---

## 🛠️ التقنيات المستخدمة
- **HTML5:** لبناء الهيكل الأساسي والنماذج (Forms).
- **CSS3:** لتنسيق الواجهة وإضافة التأثيرات البصرية (بما في ذلك مكتبة FontAwesome للأيقونات).
- **Vanilla JavaScript (ES6+):** المحرك الأساسي للمشروع، تم استخدامها بدون أي أطر عمل (No Frameworks) للتركيز على المفاهيم التالية:
  - إنشاء العناصر برمجياً (`createElement`, `appendChild`).
  - إدارة الأحداث (`EventListeners`, `Event Object`).
  - التخزين المحلي والتعامل مع صيغ البيانات (`JSON.stringify`, `JSON.parse`).
  - دوال المصفوفات المتقدمة (`filter`, `splice`).

---

## 📂 هيكل الملفات
- `index.html` : الصفحة الرئيسية التي تحتوي على واجهة الإضافة وجدول المهام.
- `Completed.html` / `To-Do.html` : صفحات مخصصة للفلترة (قيد التطوير).
- `Calendar.html` : صفحة التقويم المستقبلية.
- `CSS/` : مجلد يحتوي على تنسيقات المشروع.
- `Js/script.js` : قلب المشروع، يحتوي على منطق الجافاسكريبت مفصولاً بشكل نظيف (Clean Code) بين البيانات والـ DOM.

## 💡 دروس مستفادة من هذا المشروع
خلال بناء هذا المشروع، تم التدرب على:
1. الفصل بين "بيانات التطبيق" (Array of Objects) و "العرض المرئي" (HTML Table).
2. منع السلوك الافتراضي للنماذج باستخدام `e.preventDefault()`.
3. ربط الأحداث والدوال ديناميكياً داخل حلقات التكرار (Loops) باستخدام الـ `Index`.
4. حل مشاكل تزامن البيانات وتفادي تداخل الكلاسات برمجياً.
